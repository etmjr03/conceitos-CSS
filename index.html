Declaração CSS


  1. h1 {
  2.    color: red;
  3. }



h1 se refere ao seletor

{ } se refere a declaração

color se refere a propriedade e red se refere ao valor da propriedade.

Seletores


  • * = É o seletor global.
  • h1, h2, h3 = Seletor do tipo de elemento.
  • .red, .m-4 = Seletor de classes.
  • #box, #container = Seletor de ID.

Prioridades Style


  1. Inline quando se usa o atributo style=" " no elemento.
  2. Tag style quando faz o css dentro de uma tag <style> no arquivo HTML.
  3. Arquivo CSS quando faz o css dentro de um arquivo CSS importando no HTML.

Cascata CSS


Primeiramente devemos saber que o CSS é lido de cima para baixo, nessa ordem o atributo do seletor que terá prioridade será sempre o que está embaixo.


  • Origem do estilo = inline > tag style > arquivo style
  • Especificidade = é um valor matemático, onde, cada tipo de seletor e origem de estilo, possuem valores a serem considerados.

    0 - Seletor universal, combinações e negações pseudo-classes "(:not())".

    1 - Element Type (h1, h2), pseudo-elemento (::before, ::afeter)

    10 - Classes e atributos seletores ([type="radio"])

    100 - ID selector.

    1000 - Inline.

  • Regra !important vai sobrescrever a cascata do CSS, cuidado ao utilizar.

Shorthand CSS


O Shorthand é a "abreviação" das propriedades do CSS, ela deixa que você possa setar os valores de uma propriedade de forma simultânea, legível e economizando tempo. consulte a documentação clicando aqui!.

  1. .bg {
  2.     background-color: #000;
  3.     background-image: url(images/bg.gif);
  4.     background-repeat: no-repeat;
  5.     background-position: top right;
  6. }

Shorthand


  1. .bg {
  2.     background: #000 url(images/bg.gif) no-repeat top right;
  3. }

Distâncias Absolutas


As medidas de distâncias são fixas e não alteram seu valor, mais utilizado é o "px"


Unidades de medidas
Unidade Nome Equivalência
cm Centímetro 1cm = 96px/2.54
id Polegadas 1in = 2.54cm = 96px

Distâncias Relativas


São relativas a algum outro valor de referência, pode ser o elemento pai, ou root, ou o tabanho da tela.

Benefícios : Maior adaptação aos diferentes tipos de tela.


Unidades de medidas
Unidade Relativo a
em Tamanho da font do pai (em * pai)
rem Tamanho da font do elemento raiz (root/html)(rem * root)
vw % da viewport width
vh % da viewport height

Porcentagem CSS


Em muitos casos é tratado da mesma maneira que as medidas de distâncias e sempre será relativo a algum valor.

50% = valor do elemento pai / 2 ou sua metade
25% = valor do elemento pai / 4

Posição / position


Representa um conjunto de coordenadas 2D: top, right, bottom, left e center. Ele é usado para tipos em valores das propriedades, mas não pode ser confundido com a propriedade position:


  1. .box {
  2.      background-position: top left;
  3.      background-position: right 50%;
  4.      background-position: center;
  5. }

Box model CSS


É fundamental para fazer layouts para web, pois oferece maior facilidade para aplicar o efeito CSS.
O que é? Pode ser considerado uma caixa retangular que possui propriedades 2D e cada elemento de sua página é considerado uma caixa.
Cada elemento do HTML deve ser considerado uma caixa contendo:

  • Tamanho (width e heigth)
  • Conteúdo (coontent)
  • Bordas (border)
  • Preenchimento interno (padding)
  • Espaços fora da caixa (margin)

Box Sizing


A caixa geralmente tem seu tamanho calculada por padrão usando como referência o conteúdo presente nela content-box.
portanto para você aplicar os calculos respeitando o tamanho real do width e heigth é necessário utilizar o border-box que irá usar como referência a borda da caixa.


  1. .box {
  2.      width: 100px;
  3.      height: 100px;
  4.      padding: 0 20px;
  5.      box-sizing: content-box;
  6. }

Essa box terá o tamanho de 140px, 100 width + 20px + 20px = 140px


  1. .box {
  2.      width: 100px;
  3.      height: 100px;
  4.      padding: 0 20px;
  5.      box-sizing: border-box;
  6. }

Essa box terá o tamanho de 100px, 60px conteúdo + 20px + 20px (respeitando o tamanho de width 100) = 100px

Display Block Vs display Inline


Nesse tópico iremos entender como as caixas de comportam em relação às outras caixas, além de seu comportamento externo.

Lista de elementos e seus displays padrões
block: p, div, selection, todos os headings (h1, h2, h3...)
inline: a, strong, span, em


Diferença entre os displays
** block ** ** inline **
Ocupa toda a linha, colocando o próximo elemento abaixo desse. Elemento ao lado do outro.
width e height são respeitados. width e height não funcionam
padding, margin, border irão funcionar normalmente somente valores horizontais de margin, padding e border.

Margin


É o espaço externo entro os elementos, suas pripriedades seguem o sentido horário.

margin collapsing:

Com o display block, as margin-top e margin-bottom dos elementos abaixo e encima se juntam e dão prioridade para o maior valor.

Com o display inline, os valores de margin-top e margin-bottom dos elementos laterais são somados.


Propriedades:

  • margin-top
  • margin-right
  • margin-bottom
  • margin-left

Valores:

  • comprimentos (px, em, rem)
  • porcentagem
  • auto

Aplicando o shorthand na propriedade margin:


  1. .div {
  2.      margin: 12px 16px 10px 4px;
  3. /* 4: top, right, bottom e left */
  4.      margin: 12px 16px 0;
  5. /* 3: top, right - left e bottom */
  6.      margin: 12px 16px;
  7. /* 2: top - bottom e right - left */
  8.      margin: 12px;
  9. /* 1: aplica para todos */
  10. }

Padding é o preenchimento interno da caixa.


Propriedades:

Valores:

  • porcentagem
